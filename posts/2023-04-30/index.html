Welcome to my blog

30 Apr 2023

Introduction

So, i avoided blogging for ages and instead messaged people on various platforms or wrote to myself in files or a personal discord. So i’m starting a blog in 2023? Yes, mostly because it’s all markdown in the backend, and markdown is great to edit, for the most part üôÉ. Since messaging people my thoughts is highly inefficient and wastes their time, more likely than not, i’m gonna write to my self on this nifty blog. (And anyone who finds this and didn’t skip over this part). üëç Messaging people ideas and later realizing you are more explaining things to yourself is nothing new, it’s the good old rubber duck debugging, except with ideas. https://en.wikipedia.org/wiki/Rubber_duck_debugging

Why a blog and not a journal? Well some of the markdown files i’ll just keep private to journal, some i’ll actually turn into blog posts. Good enough, and means i can edit from anywhere where i’ve got git and an editor.

First step was to auto-generate this blog from markdown using hugo on github pages. Not because i have anything against jekyll, written in ruby. But hugo, written in go, is more familiar, i already used it on a few real projects, for my day job and if i had to deploy a new static website again i would still use hugo. ü§∑

I’ll attach the github action deploy script, i ended up stitching together, at the bottom.

This will be my blog and this was my first blog post.

Hope you enjoyed. üò∫

#https://finisky.github.io/deployhugofromprivaterepo.en/
#https://gohugo.io/hosting-and-deployment/hosting-on-github/
#
# This is a basic workflow to help you get started with Actions

name: Hugo Build & Deploy - Private to Public

  # Controls when the action will run. Triggers the workflow on push or pull request
  # events but only for the master branch
on:
  push:
    branches:
      - main

  # Allows you to run this workflow manually from the Actions tab
  workflow_dispatch:

    # Sets permissions of the GITHUB_TOKEN to allow deployment to GitHub Pages
    permissions:
      contents: read
      pages: write
      id-token: write

  # Allow only one concurrent deployment, skipping runs queued between the run in-progress and latest queued.
  # However, do NOT cancel in-progress runs as we want to allow these production deployments to complete.
concurrency:
  group: "pages"
  cancel-in-progress: false

  # Default to bash
defaults:
  run:
    shell: bash

  # A workflow run is made up of one or more jobs that can run sequentially or in parallel
jobs:
  # This workflow contains a single job called "build"
  build:
    # The type of runner that the job will run on
    runs-on: ubuntu-latest
    env:
      HUGO_VERSION: 0.111.3
    steps:
      - name: Install Hugo CLI
        run: |
          wget -O ${{ runner.temp }}/hugo.deb https://github.com/gohugoio/hugo/releases/download/v${HUGO_VERSION}/hugo_extended_${HUGO_VERSION}_linux-amd64.deb \
          && sudo dpkg -i ${{ runner.temp }}/hugo.deb
      - name: Install Dart Sass Embedded
        run: sudo snap install dart-sass-embedded
      - name: Checkout
        uses: actions/checkout@v3
        with:
          submodules: recursive
          fetch-depth: 0
      # - name: Setup Pages
      #   id: pages
      #   uses: actions/configure-pages@v3
      - name: Install Node.js dependencies
        run: "[[ -f package-lock.json || -f npm-shrinkwrap.json ]] && npm ci || true"
      - name: Build with Hugo
        env:
          # For maximum backward compatibility with Hugo modules
          HUGO_ENVIRONMENT: production
          HUGO_ENV: production
        run: |
          hugo \
          --gc \
          --minify \
          --baseURL "${{ steps.pages.outputs.base_url }}/"

      # - name: Upload artifact
      #   uses: actions/upload-pages-artifact@v1
      #   with:
      #     path: ./public

      # Deploy the Static Site to Public Repo (GitHub Pages)
      # https://github.com/JamesIves/github-pages-deploy-action
      - name: Deploy üöÄ
        uses: JamesIves/github-pages-deploy-action@v4
        with:
          token:  ${{ secrets.DEPLOYBLOG }}
          repository-name: 887/887.github.io
          branch: main # The branch the action should deploy to.
          folder: public # The folder the action should deploy.
          single-commit: true
          #commit-message: "Deploy by source"
